# 세종도서관 해외정책자료 자동수집 파이프라인 — 개발 계획서

**작성일:** 2026-03-19
**프로젝트명:** 세종도서관 해외정책자료 자동수집 시스템
**목표 산출물:** 2026년 발행 해외 정책문서 300건 (세종도서관 납품 Excel 양식)

---

## 1. 프로젝트 개요

### 1.1 목적
해외 16개국 100+ 정부·연구기관의 정책문서를 자동 크롤링하여 **필터링 → 점수 산출 → 균등 선별 → 메타데이터 보강 → 검증 → Excel 납품**까지 일괄 처리하는 자동화 파이프라인을 구축한다.

### 1.2 시스템 범위
| 구분 | 내용 |
|------|------|
| 대상 국가 | 16개국 (GB, US, SE, SG, AT, CA, NO, IT, CN, BE, IN, EU, HU, LT, MY, NZ, QA, SA 등) |
| 대상 기관 | 100+ 정부·연구기관 (sites.csv 관리) |
| 수집 기간 | 2026년 1월 1일 이후 발행 문서 |
| 최종 산출물 | 300건 선별 + 세종도서관 납품 양식 Excel (.xlsx) |

### 1.3 기술 스택
| 항목 | 기술 |
|------|------|
| 언어 | Python 3.13 |
| HTTP 클라이언트 | requests |
| HTML 파싱 | BeautifulSoup4 + lxml |
| JS 렌더링 폴백 | Playwright (Chromium headless) |
| Excel 출력 | openpyxl |
| CLI | argparse |
| 데이터 저장 | JSON 파일 기반 |

## 2. 개발 단계 및 마일스톤

### Phase 1: 기반 인프라 (1주차)

#### Task 1.1 — 사이트 관리 모듈 (`site_manager.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `Site` 데이터클래스 (22개 필드)
  - `SiteManager` 클래스 (sites.csv 파싱, 국가별/코드별 조회)
  - URL 하드코딩 수정 (9개 사이트)
- **검증:** 100+ 사이트 로드 확인, 국가별 필터 동작 확인

#### Task 1.2 — BRM 분류 매퍼 (`brm_mapper.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `TOKEN_TO_CODE1` 매핑 (17개 대분류)
  - `BrmMapper.get_brm_for_site()` — site.brm_category 파싱 → (CODE1, CODE2) 튜플 반환
  - 쉼표, 줄바꿈, ".", "및" 구분자 처리
- **검증:** BRM code.csv와 대조하여 매핑 정확도 확인

#### Task 1.3 — 기수집 필터 (`completed_filter.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `completed sites.csv` 로드 (utf-8-sig)
  - 제목 lowercase 정규화 → set 기반 중복 판정
- **검증:** 기존 수집 목록과 대조

#### Task 1.4 — 날짜 필터 (`date_filter.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `CUTOFF_DATE = 2026-01-01`
  - 7개 날짜 형식 파싱 (ISO 8601, 영문 월, 슬래시 등)
  - 날짜 없는 문서는 유보 (유지)
- **검증:** 다양한 날짜 형식 파싱 테스트

### Phase 2: 크롤러 개발 (2~3주차)

#### Task 2.1 — 크롤러 기반 클래스 (`BaseSiteCrawler`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 추상 기반 클래스 (ABC)
  - `_get()` — HTTP 요청 + 3회 재시도 + SSL/Playwright 폴백
  - `_soup()` — BeautifulSoup 파싱 + Cloudflare 감지
  - `_doc()` — 표준 문서 딕셔너리 생성
  - `save()` — JSON 저장
  - `_pw_get()` — Playwright 서브프로세스 JS 렌더링
- **검증:** 429/403/SSL/Cloudflare 에러 핸들링 시나리오 테스트

#### Task 2.2 — 영국 크롤러 (`govuk_crawler.py`)
- **상태:** ✅ 구현 완료
- **독립 구현** (BaseSiteCrawler 미상속 — GOV.UK Search API 전용)
- **내용:**
  - GOV.UK Search API 연동 (50건/페이지, 최신순)
  - `UK_POLICY_TYPES` 화이트리스트 (16개 문서 유형)
  - 내장 필터 파이프라인 (유형 → 기수집 → 날짜 → 관련성)
  - 요청 간격: 1.0초
- **검증:** API 응답 파싱, 필터링 결과 샘플 확인

#### Task 2.3 — 미국 크롤러 (`us_gov_crawler.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 24개 전용 크롤러 + 1개 범용 (`GenericHtmlCrawler`)
  - `CRAWLER_REGISTRY` 레지스트리 패턴
  - `UsGovCrawlerRunner` 통합 실행기
  - 주요 크롤러: GAO (GovInfo API), CRS (Congress.gov API), RAND (AEM), Census (AJAX) 등
  - 요청 간격: 1.5초
- **검증:** 기관별 크롤러 개별 동작 확인 (최소 상위 10개)

#### Task 2.4 — 스웨덴 크롤러 (`se_crawler.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 4개 크롤러 (GovSe, SIPRI, Timbro, 범용)
  - government.se 11개 부처 중복 방지 (`govse_crawled` 플래그)
- **검증:** government.se 중복 방지 로직 확인

#### Task 2.5 — 싱가포르 크롤러 (`sg_crawler.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 10개 크롤러 (MAS, MOM, IRAS, GovTech 등)
  - `SKIP_CODES` 건너뛰기 (EDB 로그인, IMDA 중복)
- **검증:** 403 응답 사이트 (MOH, MDDI) Playwright 폴백 확인

#### Task 2.6 — 범용 크롤러 (`generic_crawler.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `CountryGenericHtmlCrawler` — 4단계 추출 전략
    1. h2/h3/h4 > a (제목 태그 안에 링크)
    2. a > h2/h3/h4 (링크 안에 제목 태그)
    3. href 패턴 매칭 (/publications/, /research/ 등)
    4. li > a (리스트 아이템, 최소 15자)
  - 네비게이션 링크 필터 (25+ 키워드)
  - 링크 주변 날짜 추출
  - 전용 크롤러 국가 충돌 시 안내 메시지
- **검증:** AT, CA, NO, IT 등 5개 이상 국가 테스트

#### Task 2.7 — 추가 크롤러 (`crawl_additional.py`, `crawl_eprs.py`, `crawl_more.py`, `crawl_new_sites.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `crawl_additional.py` — 7개 소스 (Fed, ECB, Amnesty, HRW, WHO, UNDP)
  - `crawl_eprs.py` — 유럽의회 EPRS (날짜 윈도우 전략, PDF URL 생성)
  - `crawl_more.py` — 6개 소스 (Fed IFDP, ILO, NCES, ABARES, BIS)
  - `crawl_new_sites.py` — 3개 프리미엄 소스 (EPRS, IEA, Oxfam)
- **검증:** 각 크롤러 출력 JSON 구조 확인

#### Task 2.8 — CLI 진입점 (`main.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 6개 서브커맨드 (summary, crawl-uk, crawl-us, crawl-se, crawl-sg, crawl)
  - 공통 인자 (--site, --all, --max, --data-dir, --api-key, --force, --list)
  - 라우팅 로직 (전용 vs 범용 크롤러)
- **검증:** 서브커맨드별 실행 확인

### Phase 3: 필터링 및 선별 (4주차)

#### Task 3.2 — 관련성 필터 (`relevance_filter.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 8대 카테고리 정책 키워드 (139개)

  - 점수 산출: 제목 키워드(+0.15, 최대 0.7) + 설명 키워드(+0.05) + 문서 유형 보너스(+0.1~0.3)

  - 영어 문서 확인 (비ASCII 비율)
- **검증:** 선별 결과의 국가/기관 분포 확인, 쿼터 준수 여부

#### Task 3.4 — 선별 정리 (`fix_selection.py`)

### Phase 4: 메타데이터 보강 (5~6주차)

#### Task 4.1 — 발행일 추출 (`date_resolver.py`, `resolve_dates_v2.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `date_resolver.py` — 6단계 전략 (JSON-LD → Meta → time → 레이블 → CSS → 본문)
  - `resolve_dates_v2.py` — 간소화 버전 (상위 300건만)
  - 연도 범위 제한: 2020~2030
  - 2026+ 날짜 우선 반환
- **검증:** 날짜 추출 성공률 측정 (목표: 90%+)

#### Task 4.2 — PDF URL 추출 (`resolve_pdfs.py`, `resolve_pdfs_bulk.py`, `pdf_url_resolver.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - `resolve_pdfs.py` — 5단계 HTML 파싱 (a[href], download 텍스트, /pdf/ 경로, iframe, meta citation)
  - `pdf_url_resolver.py` — 클래스 기반 레지스트리 (GOV.UK Content API, GovInfo API, 사이트별 전용)
  - `resolve_pdfs_bulk.py` — 데이터 풀 전체 대상
- **검증:** PDF URL 유효성 (HEAD 요청), Content-Type 확인

#### Task 4.3 — 저자 추출 (`resolve_authors.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 6단계 소스 (meta author → article:author → citation_author → JSON-LD → DC.creator → CSS)
  - 정크 필터 (cookie, .com, http 등)
  - 최대 5명, 쉼표 구분
- **검증:** 저자 추출 성공률 측정

#### Task 4.4 — 요약/설명 추출 (`resolve_excerpts.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 4단계 소스 (meta description → og:description → twitter:description → 첫 <p>)
  - 최대 500자
- **검증:** 설명 추출 성공률 측정

#### Task 4.5 — 메타데이터 종합 추출 (`metadata_resolver.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 수록잡지, 권호정보, 키워드, 라이선스, ISBN/ISSN, 문서유형
  - `SITE_SERIES_MAP` (80+ 사이트 매핑)
- **검증:** 주요 필드 커버리지 확인

#### Task 4.6 — 메타데이터 보강 통합 (`enrich_selection.py`)
- **상태:** ✅ 구현 완료
- **내용:**
  - 제목 정리 (콜론/대시 정규화)
  - 키워드 추출 (140+ 패턴, 3~8개 보장)
  - 수록잡지 (`SITE_SERIES_MAP` → 기관명 폴백)
  - 권호정보 생성 ("Vol.2026, No.02")
- **검증:** 보강 전후 필드 충족률 비교

### Phase 5: 검증 및 출력 (7주차)

  - 부적격 문서 `_excluded = True` 처리

- **검증:** 납품 양식 템플릿과 대조, 하이퍼링크 동작 확인
