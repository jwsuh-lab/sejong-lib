# 세종도서관 해외 정책자료 자동수집 시스템 — 설명서

## 1. 시스템 목적

세종도서관(국회도서관 소속)에 납품할 **해외 정책자료**(영문 보고서, 정책문서, 연구논문 등)를 해외 정부·연구기관 웹사이트에서 **자동으로 크롤링·선별·가공**하여 세종도서관 제출 양식(Excel)으로 출력하는 파이프라인 시스템이다.

수작업으로는 수백 개 해외 기관 사이트를 일일이 확인해야 하므로, 이 시스템은 **수집 → 필터링 → 메타데이터 보강 → 선별 → 검증 → Excel 출력** 과정을 자동화한다.


## 2. 수집 대상

| 항목 | 내용 |
|------|------|
| **관리 파일** | `sites.csv` (→ `SiteManager`가 로드) |
| **대상 국가** | 영국(GB), 미국(US), 스웨덴(SW), 싱가포르(SI), 오스트리아(AT), 캐나다(CA), 노르웨이(NO), 이탈리아(IT), 중국(CN), 벨기에(BE), 인도(IN), EU 등 다국적 |
| **기관 유형** | 정부기관, 연구기관 (정책 싱크탱크 포함) |
| **사이트 수** | 수백 개 (기관코드 Z00001 ~ Z00704+) |
| **수집 기준일** | 2026년 1월 1일 이후 발행 자료만 수집 |
| **문서 언어** | 영문 자료 중심 (비영문 자료는 자동 필터링) |

## 5. 업무 흐름 (파이프라인)

### 전체 파이프라인 개요

```
[1단계] 크롤링    →  [2단계] 필터링    →  [3단계] 메타데이터 보강
    ↓                     ↓                      ↓
사이트별 JSON      정책유형/날짜/중복      PDF URL, 발행일,
문서 목록 수집      /관련성 필터링         저자, 발췌, 키워드 추출
                                               ↓
[6단계] 출력      ←  [5단계] 검증     ←  [4단계] 선별
    ↓                     ↓                      ↓
세종도서관 양식     날짜/PDF/언어         국가·키워드 균형
Excel 파일 생성     전수 검증             맞춰 300~500건 선별
```

---

### 5.1 [1단계] 크롤링 — 해외 기관 사이트에서 문서 목록 수집

**실행 방법:**
```bash
python main.py crawl-uk --all            # 영국 GOV.UK 전체
python main.py crawl-us --all            # 미국 전체
python main.py crawl-se --all            # 스웨덴 전체
python main.py crawl-sg --all            # 싱가포르 전체
python main.py crawl AT --all            # 오스트리아 (범용)
python main.py crawl CA --all            # 캐나다 (범용)
python main.py crawl-uk --site Z00115    # 영국 특정 기관만
```

**크롤러 유형:**

| 크롤러 | 대상 | 수집 방식 |
|--------|------|-----------|
| `GovukCrawler` | 영국 GOV.UK | GOV.UK Search API (JSON 응답) |
| `UsGovCrawlerRunner` | 미국 | 기관별 전용 크롤러 + HTML 파싱 (일부 GovInfo API) |
| `SeCrawlerRunner` | 스웨덴 | 기관별 HTML 파싱 |
| `SgCrawlerRunner` | 싱가포르 | 기관별 HTML 파싱 |
| `GenericCrawlerRunner` | 기타 국가 | 범용 HTML 크롤러 (다중 셀렉터 전략) |

**범용 크롤러 전략 (cascade 방식):**
1. `heading>link` — h2/h3/h4 안의 a 태그에서 문서 링크 추출
2. `link>heading` — a 태그 안의 h2/h3/h4에서 추출 (역순 패턴)
3. `pub-href` — URL에 publications/research/report 키워드 포함 링크
4. `list-items` — li > a 패턴으로 목록 기반 추출

**출력:** `data/` 폴더에 `{국가}_{기관코드}_{약어}_{날짜}.json` 형태로 저장

---

### 5.2 [2단계] 필터링 — 정책 관련 자료만 선별

크롤링 단계에서 자동 적용되는 4단계 필터:

#### (a) 정책 문서유형 필터 (`policy_filter.py`)
- **영국:** 화이트리스트 방식 — `policy_paper`, `research`, `official_statistics` 등 16개 유형만 유지
- **미국:** 블랙리스트 방식 — `Video` 등 명확한 비관련 유형만 제외
- **스웨덴/싱가포르:** 전부 유지 (대부분 정책 관련)

#### (b) 발행일 필터 (`date_filter.py`)
- 2026년 1월 1일 이후 발행 자료만 유지
- 날짜 없는 문서는 판별 불가로 유지
- ISO 8601, 영문 월 이름 등 다양한 날짜 형식 파싱

#### (c) 수집완료 중복 제거 (`completed_filter.py`)
- `completed sites.csv`에 이미 등록된 자료명은 제외
- 제목 기반 소문자 정규화 비교

#### (d) 관련성 스코어링 (`relevance_filter.py`)
- **키워드 매칭:** 8개 정책 분야(경제·통상, AI·디지털, 기후·에너지, 인구·복지, 안보·외교, 보건, 주거·도시, 교육) 키워드를 title/description에 매칭
- **점수 구성:** title 매치당 0.15점, description 매치당 0.05점 + 문서유형 보너스(최대 0.3점)
- **비정책 주제 배제:** 일반 구호, 개별 시설 점검, 주간 반복 게시물 등 제외
- **품질 필터:** 제목 5자 미만, description 30자 미만(제목도 짧은 경우) 제거
- **중복 제거:** 동일 title 문서 제거 (기관 재편으로 인한 복수 수집 방지)

---

### 5.3 [3단계] 메타데이터 보강 — 상세 정보 추출

#### PDF URL 추출 (`pdf_url_resolver.py`)
```bash
python pdf_url_resolver.py --country UK     # 영국 문서 PDF 추출
python pdf_url_resolver.py                  # 전체 실행
```
- **GOV.UK:** Content API → `details.attachments[]` PDF 링크
- **GAO:** GovInfo API → `download.pdfLink`
- **기타:** 랜딩 페이지 HTML에서 `.pdf` 링크 탐색

#### 발행일 추출 (`date_resolver.py`)
- JSON-LD (`datePublished`) → meta 태그 → `<time>` 태그 → 본문 정규식 순서로 추출
- Playwright 폴백 지원 (JavaScript 렌더링 필요 사이트)

#### 저자 추출 (`resolve_authors.py`)
- meta 태그 (`author`, `article:author`) → JSON-LD → byline 요소에서 추출

#### 발췌 추출 (`resolve_excerpts.py`)
- meta description → og:description → twitter:description 순서로 추출

#### 키워드/수록잡지/권호정보 (`metadata_resolver.py`, `enrich_selection.py`)
- 키워드: 제목+설명에서 정책 분야 키워드 매칭
- 수록잡지: 기관코드별 시리즈 매핑 테이블 활용
- 권호정보: 발행일 기반 자동 생성

---

### 5.4 [4단계] 선별 — 국가·키워드 균형 선별

**실행:** `python select_300.py`

**선별 로직 (3-pass 알고리즘):**
1. **Pass 1 — 키워드 최소 할당:** 부족한 분야(보건, 기후·에너지, 인구·복지 등) 최소 수량 확보
2. **Pass 2 — 국가별 쿼터:** 미국 170건, 캐나다 45건, 스웨덴/싱가포르 각 25건 등 국가별 할당량
3. **Pass 3 — 잔여 충원:** 관련성 점수 상위에서 국가 최대 쿼터(1.5배) 범위 내 충원

**추가 필터링:**
- 비영문 제목 자동 제외 (비영어 기능어 2개 이상, 비ASCII 비율 15% 이상)
- UI/네비게이션 텍스트 패턴 제외 (100+ 패턴)
- URL 내 과거 연도(/2020/ ~ /2024/) 포함 문서 제외

---

### 5.5 [5단계] 검증 — 전수 품질 점검

**실행:** `python validate_selection.py`

| 검증 항목 | 내용 |
|-----------|------|
| 날짜 검증 | 웹페이지에서 발행일 재확인, 2026년 미만 제외 |
| PDF 검증 | Content-Type, Content-Length 확인 (가짜PDF/100KB 미만 제거) |
| 언어 검증 | HTML lang 속성 + 제목 영어 여부 확인 |

---

### 5.6 [6단계] 출력 — 세종도서관 제출 양식 Excel

**실행:** `python export_selection.py` 또는 `python export_xlsx.py`

**출력 양식 (24컬럼):**

| 컬럼 | 설명 |
|------|------|
| No | 순번 |
| 국내/외 | 1 (국외) 고정 |
| 국가 | 국가코드 (GB, US, NO 등) |
| 자료명 | 문서 제목 (하이퍼링크 포함) |
| 발행처 | 기관명 |
| 저자 | 저자명 |
| 발행연도 | 4자리 연도 |
| BRM코드1/2 (2세트) | 정부기능분류체계 코드 |
| 식별번호 | 문서 식별번호 (없으면 N.A) |
| URL | 랜딩 페이지 URL |
| 첨부파일출처 | PDF 다운로드 URL |
| 수록잡지 | 시리즈명 |
| 권호정보 | 권/호 정보 |
| 발췌 | 문서 요약 |
| 키워드 | 정책 분야 키워드 |
| 라이선스 | 저작권 정보 |

**Excel 구성:**
- **시트1 "수집자료 전체"** — 전체 문서 목록 (세종도서관 양식, 필터·헤더 고정)
- **시트2 "수집 요약"** — 국가별·기관별 수집 건수 집계

## 8. 주요 기술 사항

### 의존 라이브러리
- `requests` — HTTP 요청
- `beautifulsoup4` + `lxml` — HTML 파싱
- `openpyxl` — Excel 생성
- `playwright` (선택) — JavaScript 렌더링 필요 사이트 폴백

### 속도 제한 / 예의 준수
- GOV.UK: 요청 간 1.0초 대기
- 미국 사이트: 요청 간 1.5초 대기
- User-Agent에 `SejongLibrary-Crawler/1.0 (Research Purpose)` 명시
- 429 Rate Limit 응답 시 자동 재시도 (지수 백오프)

### 데이터 안전성
- 필터링 전 원본 자동 백업 (`data/backup_원본/`)
- JSON 파일에 `crawled_at`, `filtered_at`, `original_count` 등 메타데이터 기록
- URL 보정 테이블로 사이트 URL 변경 대응 (`SiteManager._URL_FIXES`)


## 9. 전형적인 운영 사이클

1. **월초:** `python main.py crawl-uk --all` 등으로 전체 크롤링 실행
2. **크롤링 후:** `python pdf_url_resolver.py`로 PDF URL 보강
3. **선별:** `python select_300.py`로 납품 대상 선별
4. **메타데이터 보강:** `resolve_authors.py`, `resolve_excerpts.py`, `enrich_selection.py` 순차 실행
5. **검증:** `python validate_selection.py`로 전수 점검
6. **출력:** `python export_selection.py`로 세종도서관 양식 Excel 생성
7. **납품:** 생성된 `세종도서관_해외자료수집_YYYYMMDD.xlsx`를 제출
