# 세종도서관 해외 정책자료 수집 시스템 기획서

**문서 버전**: 2.2
**작성일**: 2026-03-18
**최종 수집 납품일**: 2026-03-14

---

## 1. 프로젝트 개요

### 1.1 목적

국립세종도서관의 해외 정책자료 수집 업무를 자동화하는 파이썬 기반 크롤링·선별·검증·납품 시스템이다. 35개국 450여 개 해외 정부기관·연구기관·국제기구의 정책 문서를 체계적으로 수집하고, 세종도서관 납품 양식(25컬럼 Excel)으로 출력한다.

### 1.2 배경

- 해외 정책자료는 각국 정부·연구기관 웹사이트에 산재하여 수작업 모니터링이 비효율적
- 기관별 웹사이트 구조가 상이하여 범용 + 전용 크롤러 혼합 전략 필요
- 수집 결과물의 품질 기준(발행연도, PDF 유무, 기관 균등 분배, 키워드 등)이 엄격하여 자동화된 검증 파이프라인 필수

### 1.3 핵심 지표

| 항목 | 수치 |
|------|------|
| 대상 국가 | 35개국 |
| 등록 기관 (sites.csv) | 415개 사이트 |
| 기수집 문서 (completed sites.csv) | 9,577건 |
| 전용 크롤러 보유 국가 | 4개국 (GB·US·SE·SG) |
| 범용 크롤러 지원 국가 | 31개국 |
| 신규 크롤러 추가 기관 | 8개 (EPRS, BIS, Fed FEDS/IFDP, Bruegel, ILO, Oxfam, SF Fed) |
| 수집 데이터 파일 | 193개 JSON |
| 1회 납품 목표 건수 | 250~300건 |
| 최소 기관 수 | 35개+ |
| 최소 국가/지역 수 | 10개+ |

## 4. 크롤링 전략

### 4.1 날짜 추출 공용 로직 (`extract_pub_date`)

모든 크롤러는 아래 우선순위로 발행일을 추출한다. **하드코딩 금지.**

| 우선순위 | 소스 | 설명 |
|---------|------|------|
| 1 | `<meta name="citation_publication_date">` | 학술 논문 표준 |
| 2 | `<meta property="article:published_time">` | Open Graph 표준 |
| 3 | `<meta name="DC.date.issued">` / `DC.date` / `date` | Dublin Core |
| 4 | JSON-LD `datePublished` | Schema.org 구조화 데이터 |
| 5 | `<time datetime="...">` 태그 | HTML5 시맨틱 태그 |
| 6 | 본문 텍스트 패턴 (처음 3000자) | `Month D, YYYY` / `YYYY-MM-DD` 정규식 |

추출된 연도가 2026 미만이면 해당 문서는 수집하지 않는다.

> **v2.1 변경사항**: v2.0에서는 크롤러가 `'published_date': '2026'`을 하드코딩하여 실제 2024~2025년 문서가 혼입되는 문제가 있었다. BIS 20건 중 5건(25%), ILO 3건 중 1건(33%)이 실제 pre-2026이었으며, 이를 수정하여 모든 크롤러가 실제 날짜를 추출하도록 변경하였다.

### 4.2 전용 크롤러 (4개국)

## 5. 품질 검증 시스템

### 5.1 3중 검증 체계 (`validate_selection.py`)

선별 완료 후 모든 활성 문서를 대상으로 3가지 검증을 전수 실행한다.

#### 검증 1: 날짜 검증

| 항목 | 설명 |
|------|------|
| 대상 | `published_date`가 `'2026'`(연도만)이거나 빈값인 문서 |
| 방법 | 실제 웹페이지를 다시 방문하여 메타태그/JSON-LD/time 태그/본문 텍스트에서 발행연도 추출 |
| 판정 | 추출된 연도 < 2026 → 제외 (`_relevance_score = -1.0`) |
| 부가 정보 | `_date_verified`, `_date_verified_year`, `_date_verified_by` 필드 기록 |

> **발견된 문제 사례**: BIS 2025.12월 논문 5건, ILO 2024.12월 보고서 1건, EPRS 2022.3월 문서 1건, Bruegel 2025.7월 자료 등이 `'2026'`으로 표시되어 있었음

#### 검증 2: PDF 검증

| 항목 | 설명 |
|------|------|
| 방법 | PDF URL에 HEAD 요청 → Content-Type + Content-Length 확인 |
| 가짜 PDF 판정 | Content-Type이 `text/html` → 제외 (HTML 페이지를 PDF URL로 잘못 지정) |
| 소용량 판정 | Content-Length < 100KB → 제외 (1~2페이지 인증서, 브리핑 요약 등) |
| 0바이트 처리 | Content-Type이 PDF이지만 Length=0 → GET으로 재확인 (일부 서버 특성) |

> **발견된 문제 사례**: AEI·Heritage·CNAS 등 0KB HTML 반환 14건, 35~58KB 인증서류 다수, DOI 링크가 PDF URL로 잘못 설정된 건 등

#### 검증 3: 언어 검증

| 항목 | 설명 |
|------|------|
| 비영어 단어 탐지 | 제목에 독/불/이/네/스웨/노/스페인어 기능어 2개 이상 → 제외 |
| 비ASCII 비율 | 제목의 비ASCII 문자 비율 > 15% → 제외 |
| 비ASCII + 영어 부재 | 비ASCII 3개 초과 + 영어 stopword 2개 미만 → 제외 |
| HTML lang 속성 | 웹페이지 `<html lang="...">` 확인, 비영어 + 영어 stopword < 2 → 제외 |

> **발견된 문제 사례**: 노르웨이어 문서 `Befolkningsundersøkelse 2025` (html lang="no")

#### 검증 결과 처리

- 부적격 문서: `_relevance_score = -1.0`, `_excluded = true`, `_exclude_reasons` 기록
- 적격 문서: `_validated = true` 표시
- 결과 저장: selection_400.json 즉시 반영

### 5.2 선별 단계 영어 필터 (`select_300.py`)

| 필터 | 적용 범위 | 설명 |
|------|----------|------|
| 영어 stopword 검사 | **전체 국가** | 제목에 영어 stopword 없고 정책 시작어 없으면 제외 |
| 비영어 기능어 탐지 | **전체 국가** | 32개 비영어 기능어 중 2개 이상 매칭 시 제외 |
| 비ASCII 비율 검사 | **전체 국가** | 비율 > 15% 제외 |
| 비ASCII + 영어 부재 | **전체 국가** | 비ASCII > 3개 + 영어 stopword < 2개 시 제외 |

> **v2.1 변경사항**: v2.0에서는 영어 필터가 8개 국가(nl, no, se, fi, hu, gr, ge, be)에만 적용되어 이탈리아(it), 요르단(jo), EU, 국제기구(intl) 등의 비영어 문서가 통과하는 문제가 있었다. 전체 국가로 확대 적용하고, 비영어 기능어 탐지 로직을 추가하였다.

#### 엑셀 입력 규칙 (v2.2 신규)

| 필드 | 규칙 |
|------|------|
| **제목** | PDF 자료와 동일하게 표기 (대소문자 포함). 부제가 있으면 반드시 포함. 부제는 콜론으로 구분: `주제목: 부제목` (콜론 앞 공백X, 뒤 공백O) |
| **저자** | 참여 인원 전부 기입, 콤마(,) 구분, 최대 5인까지 등록 |
| **식별번호** | ISBN과 ISSN으로 등록. 둘 다 있으면 ISBN 우선. 없으면 공란. ISBN은 하이픈 포함 번호 우선 |
| **유형** | 정책자료 / 통계자료 / 발간자료 / 보고서 / 회의자료 중 자동 분류 |
| **라이선스(CCL)** | PDF 먼저 → 웹사이트 순으로 확인. CC BY~CC BY-NC-ND 7가지 유형 + ⓒ(C). 없으면 공란 |

#### CCL(공공누리) 확인 프로세스

```
Step 1. PDF 확인 (첫 부분 또는 마지막 부분)
  ├─ CCL 마크 있음 → 해당 유형 선택 (CC BY, CC BY-NC 등)
  ├─ CCL 없음 + Copyright 있음 → C 선택
  └─ 둘 다 없음 → Step 2로

Step 2. 웹사이트(웹소개) 확인
  ├─ CCL 마크 있음 → 해당 유형 선택
  ├─ Copyright 있음 → C 선택
  └─ 없음 → 공란 (N/A)
```

CCL 유형: CC BY / CC BY-NC / CC BY-SA / CC BY-ND / CC BY-NC-SA / CC BY-NC-ND / ⓒ (C)

## 8. 실행 방법

### 8.1 표준 수집 워크플로우

```
① 기존 data/*.json에서 2026+PDF 문서 선별
   python select_300.py
     ↓
② 부족 시 신규 크롤링
   python crawl_eprs.py / crawl_additional.py / crawl_more.py
     ↓
③ 날짜 확인
   python resolve_dates_v2.py
     ↓
④ PDF URL 확인
   python resolve_pdfs.py
     ↓

   python validate_selection.py
     ↓
⑥ 균등 분배 확인 (기관당 10% 이하)
     ↓
⑦ 메타데이터 보강 + 제목 부제 포맷팅
   python enrich_selection.py
     ↓
⑧ ISBN/ISSN·유형·CCL 추출 (metadata_resolver 자동 실행)
     ↓
⑨ 저자 추출 (최대 5인) + 발췌 추출
   python resolve_authors.py
   python resolve_excerpts.py
     ↓
⑩ Excel 내보내기 (25컬럼)
   python export_selection.py
     ↓
⑪ 품질 체크리스트 확인 → 납품
```
